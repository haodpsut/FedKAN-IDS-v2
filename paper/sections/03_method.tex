The FedKAN-IDS framework instantiates Definition \ref{def:fl-ids} with each client model $f(\Theta; \cdot)$ realised as a single-hidden-layer KAN of input dimension $n_{\text{in}}$, hidden width $n_h$, and output dimension equal to the number of classes. The full architecture is depicted in Figure \ref{fig:flow} (TikZ-rendered, see Section \ref{sec:experiments} for details). Server-side aggregation is data-weighted averaging applied element-wise to the three KAN parameter sets $\{w^{(b)}, w^{(s)}, c\}$.
